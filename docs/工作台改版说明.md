# 工作台（Workspace）改版说明

## 更新日期：2025年12月24日

---

## 一、核心改动：页面职责划分

### 改动背景
原先页面职责不清晰，用户在多个页面之间来回跳转，体验混乱。

### 改动内容
明确三个核心页面的职责：

| 页面 | 职责 | 路径 |
|------|------|------|
| **悬赏详情页** | 任务信息展示、编辑、查看竞标者 | `/quest/:id` |
| **任务编辑页** | 修改任务信息（独立页面） | `/quest/:id/edit` |
| **工作台页面** | 执行任务、提交成果、审核历史 | `/workspace/quest/:id/console` |

---

## 二、Bid（竞标）流程简化

### 改动背景
Web3 场景下，用户连接钱包后平台即可获取用户地址，无需额外的密码验证步骤。

### 改动内容

**竞标提交流程：**
- 之前：填写表单 → 密码验证弹窗 → 提交成功
- 现在：填写表单 → 直接提交成功

**提交后操作：**
- 移除了「修改报价」和「撤回竞标」按钮
- 改为显示「View My Proposal」按钮，跳转到工作台管理

> 后续修改报价等操作统一在工作台页面完成。

---

## 三、任务状态显示优化

### 改动内容
页面顶部的状态标签根据实际情况动态显示：

| 状态 | 说明 |
|------|------|
| **BIDDING** | 竞标中，尚未开始执行 |
| **IN PROGRESS** | 进行中，尚未提交任何成果 |
| **UNDER REVIEW** | 审核中，已提交成果等待发布者审核 |
| **REJECTED** | 已驳回，最近一次提交被发布者拒绝 |
| **COMPLETED** | 已完成，任务成功结束 |

---

## 四、术语统一

### 改动内容

| 位置 | 原描述 | 新描述 |
|------|--------|--------|
| 工作台列表页 | Value / Budget | **Amount**（金额） |
| 工作台详情页 | Total Budget | **Total Amount**（总金额） |

> 具体含义根据任务状态和角色自动理解：竞标阶段是报价，执行阶段是结算金额。

---

## 五、界面简化

### 移除的元素
1. 任务列表中每行右侧的小箭头图标
2. 顶部的「Filter」筛选按钮
3. 工作台页面中的任务详情展开区域（改为链接跳转）

### 新增的元素
- 工作台 Task Details 区域增加「View Full Details」链接，跳转到悬赏详情页

---

## 六、发布者编辑功能

### 改动背景
编辑功能需要更大的空间，未来可能引入 AI 辅助填充，因此改为独立页面。

### 改动内容
- 编辑入口从「弹窗」改为「独立页面」
- 路径：`/quest/:id/edit`
- 页面结构与「创建任务」页面对齐

### 编辑流程
1. 点击「Edit Task」按钮
2. 进入编辑页面（4步流程：Basics → Standards → Terms → Preview）
3. 点击「Save Changes」保存
4. 返回悬赏详情页

### 可修改的字段
- 标题
- 主题/学科（下拉选择）
- 标签
- 任务描述
- 附件
- 交付物清单
- 验收标准
- 截止日期
- 交付周期

### 不可修改的字段（已锁定）
- **预算金额** — 任务发布后不可更改
- **货币类型** — 任务发布后不可更改
- **知识产权归属** — 任务发布后不可更改

> 锁定字段显示为灰色禁用状态，并标注「Locked」提示。

---

## 七、页面跳转逻辑

### 从工作台列表点击任务

| 角色 | 任务状态 | 跳转目标 |
|------|---------|---------|
| **Maker** | RECRUITING（招募中） | 悬赏详情页（显示 Edit Task + View Bidders） |
| **Maker** | IN_PROGRESS / UNDER_REVIEW / REJECTED / COMPLETED | 工作台页面 |
| **Solver** | BIDDING | 工作台页面（可修改报价） |
| **Solver** | IN_PROGRESS / UNDER_REVIEW / REJECTED / COMPLETED | 工作台页面 |

### 悬赏详情页的按钮

| 角色 | 显示按钮 |
|------|---------|
| **Maker** | Edit Task、View Bidders (数量) |
| **Solver**（未竞标） | Bid Now、View Publisher's Profile |
| **Solver**（已竞标） | View My Proposal（跳转工作台） |

---

## 八、工作台页面内容

### Task Details 区域
- 显示任务描述（截断3行）
- 显示技能标签
- 显示预算、截止日期、交付时间、IP Rights
- 提供「View Full Details」链接跳转到悬赏详情页

### 不同角色看到的操作

**发布者（Maker）：**
- 查看竞标者提交的成果
- 审核并接受/驳回提交

**竞标者（Solver）：**
- 竞标阶段：查看/修改自己的报价方案
- 执行阶段：提交工作成果
- 查看提交历史和审核结果

---

## 总结

本次改版的核心目标：

1. **职责清晰** — 悬赏详情页负责「信息展示与编辑」，工作台负责「任务执行」
2. **流程简化** — 移除不必要的验证步骤，减少页面跳转
3. **体验一致** — 统一术语、统一操作入口
4. **可扩展性** — 编辑页面独立，为未来 AI 辅助功能预留空间
